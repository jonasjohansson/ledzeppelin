# 3D Mapping (2D/3D mode) Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Give fixtures/points real XYZ and project the existing 2D composition onto the 3D-arranged rig via a fixed projection camera, so a 2D sweep travels through an arc with real foreshortening ‚Äî with 2D as the flat-front special case and every existing show unchanged.

**Architecture:** Points carry `[x,y,z]` (missing z ‚áí 0). A new pure `project3d.js` turns world XYZ + a camera into canvas UV. `pipeline.js`'s `fixtureSampleUVs` resamples a fixture's polyline by **3D arc-length** then **projects each LED** to UV; 2D fixtures keep the existing 2D path byte-for-byte. The output/daemon path is untouched ‚Äî 3D is authoring-only. See the design: `docs/plans/2026-07-01-3d-mapping-design.md`.

**Tech Stack:** Vanilla ESM, no build step. Hand-rolled projection (zero new runtime deps). Tests: `node --test "test/*.test.js"` (`node:test` + `node:assert/strict`).

**Conventions:** Test files `test/<name>.test.js`, `import { test } from 'node:test'; import assert from 'node:assert/strict';`, import from `../src/model/‚Ä¶`. Commit after every green step. Run the FULL suite before each commit (297 tests must stay green).

// src/model/project3d.js
// Hand-rolled camera projection (zero deps). World coords share today's normalized
// canvas space: x,y ‚àà 0..1 across the canvas, z in the same unit scale (z=0 = the
// canvas plane). 2D mode uses a FLAT camera that drops z, reproducing today exactly.

// A camera config: { mode: 'flat' | 'ortho' | 'perspective', pos, target, up, fov, orthoHeight, aspect }.
export function flatCamera() { return { mode: 'flat' }; }

export function project(P, cam) {
  if (!cam || cam.mode === 'flat') return [P[0], P[1]]; // exact ‚Äî no float drift for 2D regression
  return projectFramed(P, cam);
}
```

### Task 2: `project3d.js` ‚Äî perspective + ortho camera (lookAt)

import { perspectiveCamera, project } from '../src/model/project3d.js';

test('perspective: a point on the camera axis lands at centre UV', () => {
  // camera at (0.5, 0.5, 1) looking toward -z at the canvas centre; 90¬∞ fov, square.
  const cam = perspectiveCamera({ pos: [0.5, 0.5, 1], target: [0.5, 0.5, 0], fov: 90, aspect: 1 });

  assert.ok(Math.abs(u - 0.5) < 1e-9 && Math.abs(v - 0.5) < 1e-9);
});

test('perspective: equal offsets at greater depth project closer to centre (foreshortening)', () => {
  const cam = perspectiveCamera({ pos: [0.5, 0.5, 1], target: [0.5, 0.5, 0], fov: 90, aspect: 1 });
  const near = project([0.7, 0.5, 0], cam);   // depth 1 from camera
  const far  = project([0.7, 0.5, -1], cam);  // depth 2 from camera
  assert.ok(Math.abs(far[0] - 0.5) < Math.abs(near[0] - 0.5)); // farther ‚Üí nearer centre
});
```

**Step 2 ‚Äî run, expect FAIL.**

**Step 3 ‚Äî impl** `perspectiveCamera(cfg)`, `orthoCamera(cfg)`, and `projectFramed(P, cam)` using a hand-rolled basis (forward = normalize(target‚àípos), right = normalize(cross(forward, up)), trueUp = cross(right, forward)); camera-space `(cx,cy,cz)` via dot products; perspective divide `x = cx / (cz¬∑tan(fov/2)¬∑aspect)`, `y = cy / (cz¬∑tan(fov/2))`; UV `= [(x+1)/2, (1‚àíy)/2]`. Guard `cz ‚â§ 0` (behind camera) ‚Üí return the point flagged/clamped. Include small vec helpers (sub/dot/cross/normalize).

## Phase 5 ‚Äî Projection camera ‚Äî ‚úÖ DONE

Shipped: **3D placement now shapes the sampling.**

- `PROJECTION_PRESETS` / `projectionPreset(id)` / `setProjectionPreset(show, id)` in `project3d.js`:
  - **Flat (2D)** ‚Äî the default (z dropped; byte-identical with 2D ‚Äî entering 3D alone never changes the LEDs).
  - **Front** ‚Äî an ORTHO camera on the ‚àíz side framing the canvas exactly. The safe non-flat default: z=0 is EXACTLY the identity (a flat rig keeps sampling precisely where 2D put it); lifted shapes change through the 3D arc-length resample (an arch bunches toward its steep ends).
  - **Front wide** ‚Äî the same axis as a 90¬∞ PERSPECTIVE camera (canvas exactly full-frame at z=0): real depth foreshortening ‚Äî the arch's crown, farther from the camera, compresses (UVs DENSER near the apex than under Flat ‚Äî the payoff unit test; the original "a line travels through an arc differently").
- **UI** ‚Äî a "PROJECTION" row in the 3D viewport's top-left corner; changing it rebuilds the sampler, is undoable, and persists in `view3d.projectionCamera` (tagged `preset` for the active state). The camera draws as a frustum gizmo (marker + 4 corner lines) in the orbit viewport.
- **v1 limits (future work):** the gizmo is NOT draggable ‚Äî placement is preset-only (front axis, fixed position). Free camera placement, other angles (top/side), and multiple projection cameras / blending are follow-ups.
